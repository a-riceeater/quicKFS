# Security policy

Only the latest source revision is supported. Private vulnerability reporting will be enabled before a public release; until then, contact the maintainers privately rather than opening a public issue.

quicKFS is experimental. It assumes a trusted server certificate, a private network, a protected development token, and an untrusted client confined to one export root. Authentication is rudimentary; denial-of-service resistance, credential lifecycle, audit logging, and a formal protocol review are incomplete. Do not expose the prototype to the public Internet without understanding these risks.
