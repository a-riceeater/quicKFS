# Architecture

The protocol crate owns versioned wire types. `transport-quic` owns Quinn and TLS. `client-core` exposes `RemoteFilesystem`; platform adapters depend only on this interface. `server-core` owns filesystem policy and has no Quinn dependency. The Linux daemon binds transport to request dispatch.

Each operation uses an independent bidirectional stream on one long-lived QUIC connection. The client sends a 10-second keepalive, both peers use a five-minute idle timeout, and expanded connection/stream flow-control windows allow several maximum-size reads to progress concurrently. Control messages are length-prefixed Postcard frames. Read/xattr responses and write/xattr requests state their byte counts; bounded raw bytes immediately follow, avoiding embedding bulk data in serialization.

Protocol v6 adds a native directory projection: one `ListDirectoryView` returns current/parent metadata, all child metadata, complete xattr-name sets, and bounded small xattr values. The server discovers child state concurrently under a global semaphore and validates the directory revision before replying. The macOS adapter single-flights a cold directory load and serves the resulting Finder callback burst from local state rather than translating each callback into another network request. The view and its child metadata share a 30-second lifetime; mutations invalidate every local and persistent view embedding an affected node.

The export root is opened and retained as a directory descriptor. Filesystem operations walk descriptor-relative components with no-follow checks; clients never submit server paths. Opaque node IDs are keyed digests of backing device/inode identity, while the epoch/key persist in protected state outside the export so IDs survive rename, hardlinks, reconnect, and daemon restart. Known-node caches have per-connection/server-wide budgets and are released through FUSE forget; handles remain random and connection-local.

TLS protects transport and negotiates a version-specific ALPN. Server trust is selected explicitly: mutual role-separated pairing proofs, a centrally imported exact pin, a deployed private-CA bundle, or the operating-system public/managed trust policy. Exact-pin modes authenticate the leaf fingerprint; CA modes perform standard chain and server-name validation. Every mode authenticates the server before the password prompt and again on the post-prompt connection. Password records use Argon2id, authentication work/source attempts are bounded, and each account has a fail-closed write grant beneath the shared export. External certificate renewal uses private generations selected atomically. Per-user export roots, distributed-login defense, and production identity recovery remain incomplete.

The macOS adapter owns the synchronous-callback/async-runtime bridge. One multi-thread Tokio runtime creates an authenticated reconnecting/cached `RemoteFilesystem` and remains shared by all macFUSE callbacks for the mount lifetime. Stable local inode and logical-handle tables translate FUSE integers to opaque protocol IDs; callbacks cover ordinary read/write operations, xattrs/resource forks, locks, range copy/seek, special nodes, exchange/backup/volume operations, and lookup eviction. The adapter retains its own coherent directory snapshot but does not request `FOPEN_CACHE_DIR`, and it does not send synchronous kernel invalidations from inside a callback because macFUSE uses one receive loop. Capability negotiation leaves callbacks disabled when the host macFUSE ABI does not advertise them. Windows support is expected to use WinFsp later.

The persistent cache stores authenticated-namespace metadata, directory snapshots, symlink/statfs values, and integrity-checked revision-keyed ranges under an LRU byte budget. Metadata and directory state have an in-memory front tier; a bounded 256 MiB hot range tier prevents repeated disk reads and SHA-256 verification of a large block for each small FUSE read. Sequential/copy-sized fills use aligned blocks up to 16 MiB, while sub-1 MiB probes use at most a 1 MiB block. Overlapping misses are single-flighted before either persistent lookup or network fetch and share failures as well as successes. Node mutation invalidation removes both node-owned entries and every directory snapshot embedding that node in one persistent manifest transaction. The cache is deliberately read-only during disconnect and cannot cold-start. Offline writes remain future work because they require a durable mutation journal, version/permission/lock preconditions, conflict policy, and user-visible reconciliation.
