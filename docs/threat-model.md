# Threat model

Clients are untrusted and must not escape the configured root, forge node IDs or handles, exceed resource limits, or use filesystem operations before authentication. The network is untrusted and protected by TLS. First-contact trust is authenticated by a high-entropy, short-lived pairing secret transferred out of band; later connections require the pinned certificate fingerprint. Passwords are verified against Argon2id hashes after the pin is checked.

The server host, export owner, server identity state, account administration, and pairing-secret distribution are trusted. A compromised server private key permits impersonation. A compromised client trust database can redirect that client. Current connection-local login limits do not prevent distributed or reconnect-based password guessing. Traffic analysis, compromised endpoints, production recovery/revocation, per-user authorization, formal protocol analysis, and availability attacks remain out of scope for the prototype.
