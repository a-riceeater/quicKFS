# Threat model

Clients are untrusted and must not escape the configured root, forge node IDs or handles, exceed resource limits, or use filesystem operations before authentication. The network is untrusted and protected by TLS with a version-specific ALPN. The selected server-trust policy is either mutual pairing proofs, a managed exact pin, a deployed private CA, or the operating-system public/managed roots. CA modes additionally validate certificate lifetime, key usage, chain, and the requested DNS/IP name. Passwords are verified against Argon2id hashes only after server authentication. Login work, connections, requests, per-connection and aggregate node caches, and handles are bounded; incomplete requests time out.

The server host, export owner, server identity state, account administration, pairing-secret distribution, managed pin channel, and configured CA/platform roots are trusted. A compromised server private key permits impersonation when its certificate is accepted by the client's trust policy. A compromised client trust database, root store, or CA can redirect affected clients. Per-source throttling does not prevent distributed password guessing. Traffic analysis, compromised endpoints, production recovery/live-session revocation, per-user export isolation (write grants are implemented), formal protocol analysis, and large distributed availability attacks remain out of scope for the prototype.
